HIPPIFY ART
ABOUT

What did I do?

I made a website that generates a piece of art and a song snippet based on the track name and artist that a user enters. It puts the phrase of a song that is one of the most common ones in the song on the piece of art. It generates the artwork by searching for it using the words from the snippet until it finds an artwork.

Why did I do it?

I did it because I was looking through flyartproductions page that showcases similar artworks and was inspired to make their generator.

How did I do it?

I did it by asking the user to input the name of a song and optionally its artist. It finds the song and gets its ID that is used by MusixMatch. The ID of the track is used to find the snippet of the song. When the snippet is found it is turned into an array with every word representing an element in it. The words are used as terms individually to search for artworks in Rijksmuseum database. Once a piece of art is found using a particular term, the link to its image is used as as source in the code to display it. Finally, all commas and dots are removed from the snippet, it is capitalized and displayed on the artwork.

What resources did I use?

I used MusixMatch API and Rijksmuseum API. With MusixMatch I had to look up the song, find its ID and get the snippet. With Rijksmuseum I just looked for an artwork that using terms from the snippet. Links to both of the sites are listed in the bottom left corner.

What major challenges or realizations came up?

Linking JavaScript functions to HTML code and CSS code was challenging. However, a lot of helpful guides exist online. Moreover, CSS positioning was probably the most difficult aspect of the project.

Lessons learned

I learned that I should write HTML and CSS simultaneously in order to make it efficient and clean.
